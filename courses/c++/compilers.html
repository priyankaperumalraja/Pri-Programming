What are Compilers?

Have you ever thought about how exactly words in the english language are translated into signals that a computer can understand? The process is quite interesting and is one of the most important parts of how a computer works.

Here's a metaphor describing what's happening:

Imagine you are a person who speaks English and wants to communicate with a person who speaks Tamil (My family's mothertongue!). Tamil is one of the oldest and most difficult language to learn--it would take a very long time for you and everyone else who doesn't know Tamil to learn it.

Instead, you use a translator app to speak, allowing communication to continue without you having to learn the language itself.

This is the exact situation between you and the computer.

You understand a language (in this case, a programming language), that the computer cannot understand (it only knows binary language, which we'll cover later).

Here's the runthrough:

When you write code, it's formatted in what's called a programming language. This is a language that allows coders to write instructions for how a program works without typing out instructions in normal written language.

For Example: Instead of writing out "Print out the message 'Hello World'", coders can write something like "printf('Hello World')".

This formatting called "syntax" makes writing code for the coder and understanding the code for the computer much easier.
